## Overview

Transform the French-only Chrome extension into a production-ready, self-hostable language learning platform supporting all EU CEFR languages. The journey progresses from hardening infrastructure, expanding language support, adding PDF reading capabilities, establishing code quality foundations, documenting the design system, and enabling one-click deployment for non-technical users.

## Milestone Summary

**Key Decisions:**
- chrome.storage.sync for config (MV3 service workers can't use localStorage)
- ISO 639-1 two-letter language codes (standard for web APIs)
- PDF.js v4.x (v5.x requires newer Chrome features)
- Vitest over Jest (modern ESM support)
- Dependency injection for testable routes
- Dynamic imports for ES modules in content scripts
- Apache 2.0 over AGPL (enterprise-friendly)

**Issues Resolved:**
- Hardcoded localhost URLs replaced with configurable backend
- CORS security fixed (dynamic origin reflection)
- XSS vulnerabilities mitigated (textContent over innerHTML)
- Monolithic files split into modules (93% backend, 77% extension reduction)

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- None significant
